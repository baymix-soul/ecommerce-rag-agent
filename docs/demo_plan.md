# 演示计划

> 本文定义项目各阶段的演示目标和验收标准。

---

### 目标
- 完成 AGENTS.md 项目总上下文
- 完成规则体系、命令定义、记忆文件

## 阶段 1：Backend MVP

### 目标
- Spring Boot 3 项目可启动运行
- 商品数据可从 JSON 文件加载
- 健康检查接口可用
- 环境变量配置生效

### 验收标准
- [ ] Maven 项目可正常编译和启动
- [ ] `GET /health` 返回正常，含商品数量
- [ ] 日志正常输出（不包含敏感信息）
- [ ] 商品 JSON 数据可正确加载为 Product 对象列表
- [ ] 所有 API Key 通过环境变量读取，无硬编码

---

## 阶段 2：RAG Pipeline

### 目标
- 完成文本检索的最小闭环
- Embedding API 调用可用
- 向量数据库（Qdrant）可连接
- 检索 API 返回正确候选商品

### 验收标准
- [ ] EmbeddingProvider 可成功调用外部 API 获取向量
- [ ] 向量数据库连接正常，商品向量索引构建成功
- [ ] POST /api/v1/search 返回正确的 Top-K 商品
- [ ] 检索支持类目和价格区间过滤
- [ ] 降级策略可触发（关键词检索回退）

---

## 阶段 3：Streaming API

### 目标
- SSE 对话接口完整实现
- LLM 基于候选商品列表流式生成回复
- text 和 product_card 事件正确推送

### 验收标准
- [ ] POST /api/v1/chat/stream 返回 SSE 流
- [ ] text 事件包含 LLM 生成的文本增量
- [ ] product_card 事件包含正确的商品 ID 和推荐理由
- [ ] 推荐理由基于商品字段，不编造信息
- [ ] 候选商品列表为空时，LLM 回复"暂无合适商品"
- [ ] SSE 流以 done 事件正常结束
- [ ] 异常时返回 error 事件

---

## 阶段 4：Android Client

### 目标
- 原生 Android App 可连接后端
- 对话界面可输入、发送消息
- SSE 流式文本正确渲染

### 验收标准
- [ ] App 可启动并连接到后端
- [ ] 对话输入框可用，发送按钮触发请求
- [ ] SSE 文本流增量渲染到对话界面
- [ ] 流结束后显示完成状态
- [ ] 网络异常有错误提示

---

## 阶段 5：Product Card Rendering

### 目标
- 对话中的商品卡片正确渲染
- 卡片包含商品名称、价格、图片、推荐理由

### 验收标准
- [ ] product_card 事件触发商品卡片 Composable 渲染
- [ ] 卡片显示商品名称、价格、主图
- [ ] 卡片显示 LLM 生成的推荐理由
- [ ] 卡片布局适配不同屏幕尺寸

---

## 阶段 6：Enhancement

### 目标
- 多轮对话上下文记忆
- 排除已推荐商品
- 整体体验优化

### 验收标准
- [ ] 多轮对话中 LLM 能引用历史上下文
- [ ] 已推荐的商品在后续对话中不会重复推荐
- [ ] 端到端首字延迟 < 3s
- [ ] 演示用 50-100 条商品的完整导购体验
