# APK 安装指南

> 本文档说明评委如何通过 APK 快速体验项目。

---

## 1. 适用场景

- 公网后端已部署完毕
- 评委希望快速体验，无需本地部署 Docker 或 Android Studio
- 适合比赛现场快速演示

---

## 2. 安装 APK

### 2.1 真机安装

1. 下载 APK 文件（随提交材料单独提供）
2. Android 手机打开 **设置 → 安全 → 允许安装未知来源应用**
3. 点击 APK 文件进行安装
4. 打开 App

### 2.2 Android Emulator 安装

```bash
adb install app-demo.apk
```

> `app-demo.apk` 替换为实际 APK 文件名。

---

## 3. 后端地址

APK 内置后端地址：

```text
http://<PUBLIC_IP>:8080
```

或（如有域名和 HTTPS）：

```text
https://<DOMAIN>
```

> 如果评委自己部署后端，需要使用 Android Studio 修改 `AppConfig.BASE_URL` 后重新构建 APK，不建议直接使用该 APK 连接本地后端。

---

## 4. 首次启动

1. 打开 App
2. App 自动执行 demo 登录（用户名 `demo` / 密码 `demo123`）
3. 进入商品浏览页
4. 点击/长按悬浮机器人打开聊天框

## 6. 常见问题

### 6.1 App 无法连接后端

- 检查手机网络是否正常
- 检查公网后端 `/api/health` 是否可访问：

  ```bash
  curl http://<PUBLIC_IP>:8080/api/health
  ```
- 检查服务器防火墙是否放行 8080 端口
- 如果后端地址变更，需要重新构建 APK

### 6.2 TTS 没声音

- 检查手机音量是否开启
- 检查 App 内静音按钮是否关闭
- 检查 `/api/tts/speak` 是否返回有效 `audio_url`
- TTS 失败时文字回答和商品卡片仍可正常展示

### 6.3 购物车 401

- 清除 App 数据后重新启动
- App 会自动重新 demo 登录

### 6.4 语音识别不可用

- 检查麦克风权限是否已授予
- 检查设备是否支持 Android `SpeechRecognizer`
- 某些 Emulator 可能不支持 ASR，建议用真机测试

### 6.5 商品图片加载慢

- 商品图片为本地资源，首次加载可能需要时间
- 检查手机存储空间是否充足
