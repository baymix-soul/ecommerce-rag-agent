## 模块说明

### 1. Android 客户端 (Client)
- 对话窗口：文本输入，SSE 流式消费
- 商品卡片：展示 product_id、名称、价格、图片、推荐理由
- 状态管理：ViewModel + StateFlow
- 网络层：Retrofit + OkHttp（SSE 消费）
- MVP 阶段仅支持文本输入，图片输入为后续阶段

### 2. Spring Boot API 层
- RESTful API：健康检查、商品查询
- SSE：对话接口流式返回
- 中间件：CORS、日志、全局异常处理
- 暂不实现认证（后续阶段）

### 3. 意图识别模块
- 用户查询分类：商品搜索 / 属性咨询 / 商品对比 / 闲聊
- 条件抽取：品牌、价格区间、类目、属性
- MVP 阶段用规则 + Prompt 实现，不引入单独模型

### 4. RAG 检索模块
- 文本 Embedding：通过 API 调用外部 Embedding 模型
- 向量检索：ANN 近似最近邻搜索
- 结构化过滤：按类目、价格区间等元数据过滤
- Rerank：检索结果重排序
- MVP 阶段仅文本检索，多模态为后续阶段

### 5. 对话管理模块
- 多轮对话上下文存储（MVP 阶段简单内存管理）
- Prompt 构造：拼接候选商品 + 用户问题 + 对话历史
- 导购策略：追问澄清、排除已推荐

### 6. 向量数据库
- 主选：Qdrant
- 降级：关键词检索（MVP 第一阶段直接用 JSON + 关键词检索跑通闭环）
- 预留 VectorStoreService 接口，后续切换时仅替换实现类
- 商品 Embedding 向量存储（后续接入 Qdrant 时启用）

### 7. 商品知识库
- MVP：JSON 文件存储（`resources/data/products.json`）
- 数据加载：启动时加载到内存 + 向量数据库
- 后续迁移到 SQLite

### 8. LLM 服务（外部）
- Doubao-Seed-2.0-lite（兼容 OpenAI-style API）
- SSE 流式调用
- 统一 LLMClient 抽象，便于切换模型
- 超时、重试机制
