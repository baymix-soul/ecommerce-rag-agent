# review_code — 代码审查

## 用途

对指定模块或 PR 进行代码审查，输出审查意见。

## Prompt 模板

```
你现在作为本项目的 AI 编程助手，请帮我审查以下代码：

审查范围（文件/模块/PR diff）：
审查重点（可选，如性能、安全、可读性）：

在执行前请：
1. 先读取 AGENTS.md 了解项目上下文
2. 读取 .trae/rules/coding_rules.md 作为审查标准
3. 读取 .trae/rules/rag_rules.md（如涉及 RAG 模块）
4. 读取 .trae/memory/decisions.md 确认技术决策

请逐文件审查以下方面：
1. 代码风格是否符合规范（命名、缩进、行长等）
2. 是否有明显的逻辑错误
3. 是否有安全风险：
   - 是否泄漏 API Key
   - 是否硬编码凭据
   - 是否存在注入漏洞
   - 日志是否输出敏感信息
4. 是否有性能隐患（N+1 查询、未使用缓存等）
5. 异常处理是否完善
6. API 返回结构是否稳定
7. 是否遵守 .trae/rules/permission_rules.md 中的权限分级

输出审查报告，包含：
- 严重问题（必须修复）
- 建议改进（可选的优化建议）
- 正面反馈（做得好的地方）
```
