# fix_bug — 修复 Bug

## 用途

定位并修复项目中存在的 Bug。

## Prompt 模板

```
你现在作为本项目的 AI 编程助手，请帮我修复以下 Bug：

Bug 描述：
复现步骤：
预期行为：
实际行为：
相关日志或错误信息：

在执行前请：
1. 先读取 AGENTS.md 了解项目上下文
2. 读取 .trae/memory/decisions.md 避免引入与已有决策冲突的修复
3. 搜索相关代码，定位 Bug 根因

请完成以下操作：
1. 分析 Bug 根因，用文字描述清楚
2. 实施修复（小步改动，不连带修改无关代码）
3. 如果项目有编译命令，运行编译检查
4. 如果项目有测试命令，运行相关测试验证修复
5. 确认修复不引入新问题
6. 更新 .trae/memory/changelog.md 记录修复内容
7. 列出修改的文件和修复要点

注意：
- 修复范围最小化，不连带修改无关代码
- 如果 Bug 涉及多个模块，按依赖顺序修复
- 复杂 Bug 先在 .trae/memory/todo.md 中记录分析过程
- 不修改 API 返回格式（除非经用户明确授权）
```
