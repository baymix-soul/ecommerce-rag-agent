# 权限与安全规则

## 操作权限分级

### A. 默认允许的操作

无需用户授权，可直接执行：

- 新建文档（`.md`）和设计文档
- 新建当前任务相关的代码文件
- 编辑当前任务明确相关的文件（小步改动）
- 添加代码注释
- 新增测试文件
- 新增 `.env.example` 模板项（仅占位符）
- 新增 `.gitignore` 规则
- 读取项目中的任何文件

### B. 必须经用户明确授权的操作

执行前必须向用户确认并获得授权：

- 删除文件
- 重命名文件或目录
- 大规模重构代码
- 修改依赖版本（`pom.xml`、`build.gradle` 等）
- 安装新依赖
- 修改 API 返回格式
- 修改核心 Prompt 模板
- 修改 RAG 检索策略
- 修改数据库结构
- 执行 `git push`
- 执行 `git reset` / `git rebase` / `git clean`
- 修改 `.env` 文件
- 覆盖 `AGENTS.md` 或 `.trae/rules/` 下的规则文件

### C. 绝对禁止的操作

任何情况下不得执行：

- 写入真实 API Key、Token、密码到代码、注释、日志或文档
- 将 `.env` 提交到版本控制系统
- 编造已实现的功能或虚构完成进度
- 编造商品名称、价格、库存、优惠、功效
- 删除用户代码后直接重写（应先征求意见）
- 引入不必要的复杂框架或过度设计

## API Key 与凭据管理

1. `.env` 文件不提交到版本控制
2. `.env.example` 只包含占位符（如 `your_api_key_here`），不包含真实值
3. 代码中通过环境变量读取配置，不允许硬编码
4. 日志输出中禁止打印 API Key 或 Token
5. 不在 README 或文档中暴露真实凭据
6. 所有外部 API 调用的 Key 必须从环境变量或安全配置源读取

## 提交前敏感信息检查清单

每次提交代码前必须逐项确认：

- [ ] `.env` 不在 Git 暂存区
- [ ] 代码中无硬编码的 API Key / Token / 密码
- [ ] 日志配置不会输出敏感信息
- [ ] `.gitignore` 包含 `.env` 及所有敏感文件
- [ ] 无包含真实凭据的测试数据
