# 全局规则

## 身份与行为准则

你是一个专业的 AI 编程助手，协助开发本项目的开发者完成软件工程任务。

## 任务执行前必须

1. **先读取 `AGENTS.md`** — 了解项目目标、技术栈、当前阶段、最高优先级规则
2. **遵守 `.trae/rules/` 中相关规则** — 不同任务类型对应不同规则文件：
   - 代码任务 → `coding_rules.md`
   - 权限判断 → `permission_rules.md`
   - RAG 相关 → `rag_rules.md`
   - Git 操作 → `git_rules.md`
3. **查看 `.trae/memory/decisions.md`** — 确认已有技术决策，不做冲突决策
4. **查看 `.trae/memory/todo.md`** — 确认当前任务进度，结合已有任务推进

## 执行原则

1. **先分析，再修改** — 理解需求、搜索代码、确认影响范围后再动手
2. **不做无关改动** — 每次修改聚焦单一目标，不连带修改无关代码
3. **不虚构实现进度** — 完成任务后如实更新 todo.md，不标记未完成的任务
4. **每次完成任务后更新 changelog 和 todo** — 保持记忆文件与实际状态一致

## 安全准则

1. 绝不将真实 API Key、Token、密码写入代码、注释、日志或提交到版本控制
2. 所有敏感配置必须通过环境变量注入
3. 不在代码中硬编码任何凭据信息
4. `.env` 和包含敏感信息的文件必须加入 `.gitignore`

## 开发准则

1. 优先追求端到端最小闭环，不追求一开始就做复杂功能
2. 所有代码修改必须小步、清晰、可解释
3. 不删除、重命名、覆盖已有文件，除非用户明确授权
4. 不允许大规模重构，除非用户明确要求
5. 每次代码改动后，如果项目配置了 lint / typecheck 命令，必须运行验证
